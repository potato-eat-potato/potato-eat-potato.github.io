Tool: Compare domains by monotonicity

One-Sentence Summary

Sandwich an awkward domain between two computable ones and transport the estimate across the inclusion — the lazy person’s sharp bound.

What Problem Does This Solve?

Estimates on irregular domains (punctured boxes, boxes with holes, non-round regions) where exact formulas exist only for balls and boxes.

Input

Domain inclusion \(D' \subset D\) and a monotone quantity.

The Monotonicity Inventory (memorize the directions)

Quantity Direction under \(D' \subset D\)
Killed Green function \(G_{D'}(x,y)\) \(\le G_D(x,y)\) (fewer visits before exit)
DGFF variance \(\mathrm{Var}_{D'}(\varphi_x) = G_{D'}(x,x)\) \(\le\)
Exit time \(\mathbb{E}_x \tau_{D'}\) \(\le\)
\(R_{\mathrm{eff}}\) to the boundary network version: see Rayleigh monotonicity
\(\mathbb{P}_x(\text{exit } D' \text{ before hitting } A)\), \(A \subset D'\) needs care — not always monotone; check each time

Output

Two-sided bounds: e.g. for any \(D\) with \(B_r(x) \subset D \subset B_R(x)\): \[\tfrac{2}{\pi}\log r + O(1) \le \mathrm{Var}(\varphi_x) \le \tfrac{2}{\pi}\log R + O(1).\] For “round enough” domains the sandwich is sharp to \(O(1)\) — often all I need.

What It Converts

  • from: geometry-specific hard analysis
  • to: one inclusion diagram + a known formula on balls/boxes

Core Idea

Coupling: run the same walk, kill it earlier in the smaller domain — every visit-count in \(D'\) is dominated pathwise. For the field, the Gibbs–Markov decomposition (Use Gaussian conditioning as Markov property) writes \(\varphi_D = \varphi_{D'} + \text{independent harmonic part}\), so variances add — monotonicity with an exact error term. That upgrade (inequality → decomposition) is the move behind Boundary-to-bulk comparison.

When Should I Think of This?

  • First move on any perturbed-domain problem: what do the inscribed and circumscribed boxes already give?
  • To sanity-check any claimed estimate: does it respect the inventory directions?

Where This Appears in My Path

Constantly in the punctured box question: the punctured domain sits between “tiny box around the hole removed” and “full box,” and the first nontrivial fact I derived there was which comparisons are lossless.

Current Understanding

Solid; the inventory table is battle-tested.

Open Confusions
