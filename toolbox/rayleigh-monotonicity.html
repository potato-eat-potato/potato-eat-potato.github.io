Theorem

Theorem: Rayleigh monotonicity

Statement

If edge resistances increase (\(r_e \le r_e'\) for all \(e\)), then \(R_{\mathrm{eff}}(a,z) \le R'_{\mathrm{eff}}(a,z)\). Corollaries: shorting (gluing vertices) can only decrease \(R_{\mathrm{eff}}\); cutting (removing edges) can only increase it.

Intuition

Making every wire worse cannot improve the network. Obvious physically — the content is that “effective” resistance, a global solve, inherits the edgewise order.

When to Use

Comparison without computation: modify the network into something solvable and keep track of the direction. Mnemonic (from the tool note): short for lower bounds, cut for upper bounds.

What It Converts

A quantitative question about an unsolvable network into a qualitative comparison with a solvable one.

Proof Sketch

One line from Thomson principle: \(R'_{\mathrm{eff}} = \min_\theta \sum_e \theta(e)^2 r'_e \ge \min_\theta \sum_e \theta(e)^2 r_e = R_{\mathrm{eff}}\) — the same competitor flows, evaluated with larger weights. (Monotonicity is a corollary of a variational characterization; this pairing is a pattern worth remembering in general.)

Typical Failure Modes

  • Direction errors — the perennial bug. Re-derive from “cut = \(r_e \to \infty\), short = \(r_e \to 0\)” every time.
  • Monotonicity is for \(R_{\mathrm{eff}}\) between fixed terminals; quantities like exit distributions are not monotone this way.
  • Probabilistic consequences need the dictionary factor: escape probability involves \(c(a)\), which surgery may also change.

Related Tools

  • Use Rayleigh monotonicity by shorting or cutting (action version, with the \(\mathbb{Z}^2\) and \(\mathbb{Z}^3\) showcases)
  • Compare domains by monotonicity (the same philosophy, one level up)
