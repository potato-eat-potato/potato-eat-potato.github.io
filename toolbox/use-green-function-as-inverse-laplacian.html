Tool: Use Green function as inverse Laplacian

One-Sentence Summary

Treat \(G_D\) as the literal matrix inverse of \(-\Delta\) on the domain — solve discrete PDEs by summing against a kernel, and read walk facts as linear algebra.

What Problem Does This Solve?

Any discrete Poisson problem: find \(u\) with \(-\Delta u = f\) in \(D\), \(u = 0\) on \(D^c\).

Input

The domain \(D\), the source \(f\), and the Killed Green function \(G_D\).

Output

\[u(x) = \sum_{y \in D} G_D(x,y)\, f(y),\] unique by the Maximum principle. Probabilistically: \(u(x) = \mathbb{E}_x\big[\sum_{k < \tau} f(X_k)\big]\) — solutions of Poisson equations are expected accumulated rewards.

What It Converts

  • from: a difference equation (analysis)
  • to: a kernel sum (linear algebra) ↔︎ an occupation-time functional (probability)

Core Idea

\(-\Delta\) with Dirichlet boundary is an invertible positive matrix; \(G_D\) is its inverse; positivity of the inverse is the probabilistic miracle (entries count visits, so they’re \(\ge 0\)). Three languages, one identity. The Gaussian upgrade — a positive-definite inverse can serve as a covariance — is exactly Express DGFF covariance by Green function.

When Should I Think of This?

  • An equation has the shape “Laplacian of unknown = known.”
  • I see \(\mathbb{E}\big[\sum_{k<\tau} (\cdots)\big]\) — recognize an inverse Laplacian and switch to analysis.
  • I need positivity or monotonicity of a solution: inherit it from the kernel.

Typical Use Cases

  • Expected exit time: \(-\Delta u = 1\) ⇒ \(u(x) = \sum_y G_D(x,y)\).
  • Occupation-time identities in the Zeitouni project.
  • Setting up the covariance computations in DGFF on punctured box.

Where This Appears in My Path
