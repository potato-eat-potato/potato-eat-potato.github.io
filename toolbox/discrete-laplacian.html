Object: Discrete Laplacian

One-Sentence Summary

The operator that says “compare a function to the average of its neighbors” — the grammar in which random walks, harmonic functions, networks, and the DGFF all speak.

Definition and VAULT CONVENTION

This vault’s default (probabilistic normalization): \[\Delta f(x) = \frac{1}{2d}\sum_{y \sim x}\big(f(y) - f(x)\big) = (P - I)f(x).\]

Other conventions in the wild — convert before mixing sources:

Convention Operator Used by
Probabilistic (vault default) \(\frac{1}{2d}\sum_{y\sim x}(f(y)-f(x))\) Lawler, Zeitouni notes
Graph Laplacian \(\sum_{y\sim x}(f(y)-f(x))\) spectral graph theory; factor \(2d\)
Network / weighted \(\frac{1}{c(x)}\sum_y c_{xy}(f(y)-f(x))\) Lyons–Peres

Consequences of the choice propagate to Green function (factor \(2d\)) and the DGFF density. Getting this wrong is my single most likely source of constant errors.

Intuition

\(\Delta f(x) > 0\): \(f\) at \(x\) sits below its neighborhood average — the walk “drifts uphill in expectation.” Harmonic (\(\Delta f = 0\)) = mean-value property = fair game for the walk = no net current.

Equivalent Interpretations

  • Generator of the SRW: \(\mathbb{E}_x[f(X_1)] - f(x)\).
  • Negative gradient of Dirichlet energy: critical points of \(\mathcal{E}\) with fixed boundary are harmonic.
  • Kirchhoff’s node law for the network with potential \(f\).

Why This Object Is Central to My Probability Route

Every object in my toolbox is defined through \(\Delta\): harmonic functions (\(\Delta h = 0\)), Green function (\(-\Delta G = \delta\)), potential kernel (\(\Delta a = \delta_0\)), DGFF (Gaussian with \((-\Delta)^{-1}\) covariance). It is the root node of the whole dependency graph.

Tools That Use This Object

  • Use Green function as inverse Laplacian
  • Turn hitting probability into harmonic function
  • Express DGFF covariance by Green function

Where I Encountered This

Everywhere; first consciously in Honors Probability (martingale problems), then operationally in the Zeitouni project.
