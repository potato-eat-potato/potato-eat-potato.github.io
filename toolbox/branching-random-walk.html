Object: Branching random walk

One-Sentence Summary

Particles that branch and move independently — the cleanest model where “exponentially many correlated trajectories” can be analyzed exactly, and the toy model for everything log-correlated.

Definition

Start with one particle at \(0\). Each generation, every particle splits into offspring (mean \(m > 1\)) which each jump by an independent copy of a step distribution. $M_n = $ position of the rightmost particle at generation \(n\).

The Identity That Makes It Computable

Many-to-one lemma (simple version): for the count over generation-\(n\) particles, \[\mathbb{E}\Big[\sum_{|v| = n} f(S_v)\Big] = m^n\, \mathbb{E}[f(S_n)],\] where \(S_n\) is an ordinary random walk. First moments of BRW = (growth factor) × (single-walk computation). This is why Use first moment method for maximum upper bound + Cramer theorem immediately give the speed.

Headline Behavior

  • Speed: \(M_n / n \to v^*\) where \(I(v^*) = \log m\) (\(I\) = rate function of the step; Hammersley–Kingman–Biggins). Balance: \(e^{n\log m}\) particles vs. \(e^{-nI(v)}\) cost — pure Entropy versus energy.
  • Log correction: \(M_n = nv^* - \frac{3}{2\theta^*}\log n + O_P(1)\) (Bramson for BBM; Addario-Berry–Reed, Hu–Shi for BRW). The \(\frac{3}{2}\), vs. \(\frac{1}{2}\) for independent walkers, is the price of the tree correlations.
  • Convergence in law of the recentered max (Aïdékon) — future reading.

Why This Object Is Central to My Probability Route

BRW is the skeleton key for the 2D DGFF: the multiscale decomposition exhibits the field as approximately a BRW across \(\log N\) dyadic scales, and the extremal behavior matches (\(\frac34\log\log\) ↔︎ \(\frac32\log\)). Understanding BRW extremes is the entry ticket to log-correlated fields.

Tools That Use This Object

  • Use first moment method for maximum upper bound
  • Use second moment method for existence lower bound (needs Bramson’s truncation — vanilla second moment fails)
  • Use Cramer transform for large deviation speed
  • Use multiscale decomposition

Common Normalizations or Conventions

Binary branching + Gaussian steps is my default sandbox; general offspring/step laws add integrability conditions I should not quote casually.

Where I Encountered This
